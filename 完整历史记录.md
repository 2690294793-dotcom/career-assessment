## 一、项目概览

自在生涯工作室是一个面向大学生的职业测评与咨询系统，由【账号1】完整构建，【账号2】和【当前新账号】持续迭代。

### 1.1 整体架构

纯前端多页面应用，无构建工具，无框架依赖，每个页面为独立 HTML 文件，内联 CSS 和 JavaScript，通过浏览器直接打开运行。数据层采用 **Supabase 云数据库 + localStorage 本地存储双轨制**，Supabase 不可用时自动降级到本地存储。

**三层架构**:
- **页面层**: 根目录下所有 HTML 文件，页面间通过 `window.location.href` 跳转，URL 参数传递数据
- **逻辑层**: `js/` 目录下 JavaScript 文件，提供数据库配置、预设数据和端侧逻辑
- **资源层**: `css/` 提供全局共享样式，`img/` 提供图片资源

**关键文件**:
- `js/config.js`: Supabase 统一配置，含数据库操作函数（saveSubmission、getAppointments、createAppointment、saveConsultationNotes 等）
- `js/articles-data.js`: 26篇预设公众号文章，作为 Supabase 兜底数据
- `js/student.js`: 霍兰德职业兴趣类型常量（CAREER_TYPES）
- `js/teacher.js`: 教师端常量（TEACHER_PASSWORD、TEST_INFO）
- `database.sql`: Supabase 建表脚本与 RLS 策略

## 四、关键技术决策记录

### 4.1 Supabase 数据库设计
- **submissions** 表：提交记录表，存储测评结果和表单数据
- **appointments** 表：预约信息，含 feedback_tags/feedback_text/feedback_at/has_feedback
- **mentor_schedule** 表：`specific_date DATE`（具体日期，非周几）+ start_time/end_time
- **mentor_settings** 表：`booking_open BOOLEAN`（仅此一个开关字段）
- **articles** 表：文章信息，支持微信HTML内容
- **RLS策略**：appointments有SELECT/INSERT/DELETE；mentor_schedule/mentor_settings有SELECT/INSERT/DELETE

### 4.2 GitHub 推送方式
- 国内 git push 不稳定，改用 GitHub Contents API（PUT /repos/{owner}/{repo}/contents/{path}）
- 需用户提供 Personal Access Token（fine-grained token，勾选 Contents Read and Write）
- token 用完即删（安全措施）
- 推送脚本：`push-files.js`（支持多文件批量推送）
