5分钟配置指南

按照以下步骤配置 Supabase 云数据库，让学生的测评数据自动同步到教师端

为什么要配置？
目前学生端的数据只能保存在浏览器本地，无法同步到教师端。配置 Supabase 后，数据会存到云端， 学生提交后教师立即就能看到。Supabase 免费版提供 500MB 存储和 5万次/月访问量，完全够用， 而且链接永久有效不会过期。

1注册 Supabase 账号

  1. 打开 https://supabase.com
  2. 点击右上角 Start your project
  3. 使用 GitHub 账号或邮箱注册（完全免费）

2创建新项目

  1. 登录后点击 New Project
  2. 填写项目名称，如 career-assessment
  3. 设置数据库密码（记住它，但代码中不需要用到）
  4. 选择离你最近的区域（如 Northeast Asia - Singapore）
  5. 点击 Create new project，等待约 2 分钟初始化完成

3创建数据库表

  1. 在左侧菜单点击 SQL Editor
  2. 点击 New query
  3. 将以下 SQL 完整复制粘贴到编辑器中：
-- 创建提交表 CREATE TABLE IF NOT EXISTS submissions ( id BIGSERIAL PRIMARY KEY, name TEXT NOT NULL, school TEXT DEFAULT '', student_id TEXT NOT NULL, class_name TEXT NOT NULL, test_type TEXT NOT NULL DEFAULT 'holland', test_name TEXT, answers JSONB NOT NULL DEFAULT '{}', scores JSONB NOT NULL DEFAULT '{}', result_code TEXT NOT NULL DEFAULT '', result_summary TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- 开启行级安全 ALTER TABLE submissions ENABLE ROW LEVEL SECURITY; -- 允许学生提交数据 CREATE POLICY "允许学生提交" ON submissions FOR INSERT WITH CHECK (true); -- 允许教师查看数据 CREATE POLICY "允许教师查看" ON submissions FOR SELECT USING (true); -- 允许教师删除数据 CREATE POLICY "允许教师删除" ON submissions FOR DELETE USING (true); -- 创建索引 CREATE INDEX IF NOT EXISTS idx_submissions_created_at ON submissions(created_at DESC); CREATE INDEX IF NOT EXISTS idx_submissions_class ON submissions(class_name); CREATE INDEX IF NOT EXISTS idx_submissions_school ON submissions(school);
  1. 点击右下角 Run 按钮，看到 "Success" 表示建表成功

4获取 API 密钥

  1. 在左侧菜单点击 Project Settings（齿轮图标）
  2. 点击 API
  3. 找到以下两项并复制：
  • Project URL：形如 https://xxxxx.supabase.co
  • anon public key：一长串字母数字（不是 service_role key）
注意：请复制 anon public 的 key，不要复制 service_role 的 key。 anon key 是公开安全的，专为前端使用设计。

5填入配置文件

打开项目中的 js/config.js 文件，替换以下内容：

① Supabase 连接信息（约第31-32行）：

const SUPABASE_URL = "https://你的项目ID.supabase.co"; const SUPABASE_KEY = "你的anon_public_key";

② 学校名称（约第24行）：把 SCHOOL_NAMES 改成你们学校名称，学生填写的学校名与此匹配才能查看完整报告

const SCHOOL_NAMES = "惠州学院"; // 改成你们学校

保存文件，重新打开页面即可。学生提交后，教师端就能实时看到数据了！

✅

配置完成！

学生端提交的数据现在会自动同步到教师端

打开学生端 打开教师端

❓ 常见问题

Q: 配置后学生提交提示失败？
检查 config.js 中的 URL 和 KEY 是否正确，确保没有多余空格。打开浏览器控制台（F12）查看具体错误信息。
Q: 教师端看不到数据？
确认 SQL 已成功执行（在 Supabase → Table Editor 中能看到 submissions 表）。确认 RLS 策略已创建。
Q: 教师端密码是什么？
默认密码是 teacher123，可以在 js/teacher.js 第 6 行修改。
Q: 链接会过期吗？
Supabase 免费版项目不会过期，链接永久有效。但项目如果超过 7 天完全无活动会被暂停，重新登录即可恢复。
Q: 数据安全吗？
目前为方便演示，数据读写是开放的。如需更安全，可在 Supabase 中添加 API 密钥验证。教师端有密码保护。
